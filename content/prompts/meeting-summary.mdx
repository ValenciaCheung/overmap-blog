prompt: |
  你是一位资深 Chief of Staff,擅长把杂乱的会议笔记 / 转写整理成清晰可执行的纪要。

  请把下面的会议内容整理成以下结构:

  ## 会议元数据
  - 时间 / 参与人 / 议程主题(从原文推断)

  ## TL;DR(≤3 句)
  这次会议最关键的一件事是什么

  ## 决策(Decisions)
  - 用"团队决定..."句式
  - 每条带责任人(原文里出现过的人名)

  ## 待办(Action Items)
  用表格:

  | 负责人 | 任务 | 截止 |
  |--------|------|------|

  - 必须有责任人(没明确的话标 "TBD")
  - 截止日期没说就标"未定"
  - 不要发明任务,只总结原文真的提到的

  ## 阻塞 / 风险(Blockers)
  - 如果有未解决问题,列出来
  - 标注谁需要去解锁

  ## 下次议程候选
  - 推断出这次没讨论完、应该下次接着聊的内容

  ## 引用片段
  - 摘 2-3 句原文的关键引用,用引号

  约束:
  - 不要复述全部对话内容,只提炼决策点
  - 客观语气,不评判
  - 中英文混合时,人名保持原文,术语跟原文一致

  会议内容:
  {{在这里粘贴会议转写 / 笔记}}

## 用法

跟 Granola / Otter / Fireflies 等会议转写工具配合最好:

1. 会议结束 → Granola 自动转写 → 复制全文
2. 把转写粘到 `{{...}}` 位置
3. 让 Claude 跑这个 prompt → 得到结构化纪要
4. 直接发 Slack channel

## 改写思路

- **一对一 1:1** → 把 "决策 / 待办 / 阻塞 / 下次议程" 改成 "成长话题 / 反馈 / 行动项 / 下次主题"
- **客户访谈** → 改成 "客户痛点 / 解决方案讨论 / commitment / 下一步"
- **设计评审** → 改成 "评审意见 / 待修改项 / 同意发布 / 风险点"

## 坑点

- Granola 已经自带这种结构化输出,只在你想要不同结构时用这个 prompt
- 1 小时会议生成的纪要应该 < 500 字,长了说明你让 AI 复述了
- 决策点没明确的话,**主动追问发起人**,不要 AI 替你猜
