prompt: |
  你是一位独立 AI 产品测评作者，正在写一篇横向对比文章，目标读者是想做选型决策的开发者 / 设计师 / PM。

  请按以下结构组织：

  ## 1. 背景（200 字）
  这几款产品分别解决了什么问题，为什么放在一起对比

  ## 2. 对比维度表
  用 Markdown 表格，行是产品，列至少包含：
  - 定价（最低能用上的方案）
  - 核心能力
  - 适用场景
  - 最大限制 / 缺点
  - 学习曲线
  - 我推荐的程度（1-5 ⭐）

  ## 3. 场景推荐
  分 3-5 个典型场景（如 "个人用户写代码"、"团队做内容运营"、"重度科研工作流"），每个场景明确推荐哪一款 + 理由

  ## 4. 谁应该避开
  每个产品列出 "不适合谁"，包括硬限制和软体验问题

  ## 5. 一句话结论
  没有标准答案，但如果非要选一个 default，会选哪个

  约束：
  - 不要夸张，不要写广告软文，直接 trade-off
  - 提到的能力都要可验证（举例 / 截图 / 引用文档）
  - 不要列所有功能，只列影响决策的差异
  - 中英文产品名第一次出现时给中文释义

  对比的产品：
  {{产品 1：xxx}}
  {{产品 2：xxx}}
  {{产品 3：xxx}}

## 用法

把 `{{...}}` 替换成具体产品。每个产品建议给一个简短描述（一句话定位 + 官网 URL），避免模型用过时的认知。

## 适合什么场景

- 选型类博客（"Claude vs ChatGPT vs Gemini 谁更适合写代码"）
- B 端售前材料
- 公众号 / 小红书的 "5 款 xx 神器对比" 选题

## 改写思路

- **更技术** → 在维度表里加 "API 价格 / 上下文长度 / 多模态支持"
- **更运营向** → 维度加 "免费额度 / 中文质量 / 中国大陆访问"
- **更短** → 缩到 3 个维度，结论提前

## 坑点

- 模型对最新版本（最近 3 个月内发布的）认知不一定准，**自己核实价格和功能**
- "推荐程度" 这种主观打分不要全信，让模型给理由，自己判断
- 涉及国内访问的，让模型给 "境内可访问 / 需要科学上网" 这一列
