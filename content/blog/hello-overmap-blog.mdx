## 为什么要做这个站点

过去几年，AI 工具像潮水一样涌出来。我们作为产品设计师 / 全栈开发者 / 内容生产者，每天都在切换不同的工作流：

- 在 **Cursor** 里跟 Claude 一起写代码
- 在 **v0** 里把 UI 草图变成 React 组件
- 在 **Midjourney** 里调风格、出概念图
- 在 **ElevenLabs** 里把脚本读成视频配音
- 在 **Notion / Obsidian** 里沉淀知识

但有几个问题始终没解决：

1. 优质工具散落在几十个 newsletter / Twitter / 公众号里，**没一个干净的去处**；
2. 我自己拍的视频、写的文章、调试出的 Prompt **没集中地方分发**；
3. GitHub 上每天有顶级开源项目冒出来，**没人帮我做"中文 + 编辑视角"的精选**。

参考过 [优设 (uisdc.com)](https://www.uisdc.com)、[aixzd.com](https://aixzd.com)、[Toolify](https://www.toolify.ai)、[Agent Skills Hub](https://agentskillshub.top) 之后，我们决定把这几件事合到一个站点里做。

## Overmap Blog 提供什么

- **AI 工具库** — 编辑挑选过的工具卡片，每张卡片就是一个外链，可分类、可搜索；
- **博客** — 我们视频内容的文字版 + Tech 圈使用 AI 的技巧；
- **Skills Hub** — GitHub 上高 star 的 Agent / 框架 / Prompt 仓库聚合，按"工程师视角"分类；
- **提示词库** — 经过实测的 Prompt，一键复制，给出适用模型、改写思路、坑点。

## 跟 Overmap 主站什么关系

[overmap.org](https://overmap.org) 是我们做的 AI API 网关产品（基于 new-api 二次开发），统一聚合 40+ 上游模型，自带账号 / 计费 / 限流后台。

这个 Blog 站点是它的二级域名内容生态：

```
overmap.org           — 产品本体（API 网关 / Dashboard）
overmap.org/docs      — 产品文档（VitePress）
blog.overmap.org      — 这里（内容 / 工具 / Hub）
```

视觉上沿用同一套设计系统（`radix-nova` 风格 + Public Sans 字体 + oklch 蓝灰冷色调），让你切换站点时不会觉得"换了个地方"。

## 想跟我们一起做

- 在 [GitHub](https://github.com/overmapai) 给 Issue / PR；
- 推荐你日常在用的 AI 工具或仓库，我们会人工筛选后收录；
- 投稿提示词或工作流文字稿，作者会署名。

接下来这个 Blog 会保持**每周更新**节奏，先把工具库、Skills Hub、提示词的种子内容补到 50+。
