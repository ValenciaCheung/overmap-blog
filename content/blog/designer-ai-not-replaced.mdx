> 这篇是写给设计师的 —— 看完应该不焦虑,但需要重新规划自己的工作组合。

## 一句话总结

**AI 替代不了"决策",但替代了大部分"产出"。** 设计师的价值正在从"我能做出 X"转移到"我能判断什么 X 值得做"。

## 5 个具体环节

### 1. 概念探索 → AI 100% 接管
以前出 8 个 mood board / 风格候选要 1-2 天,现在 Midjourney 30 分钟出 100 个。

- **替代的是**:"我也来试试这种风格" 的劳动
- **没替代的是**:"哪个方向最贴合 brand?" 的判断
- **设计师怎么用**:学会快速过滤 100 个候选,定 3 个,而不是死磕一个

### 2. UI mockup → AI 70% 接管
Galileo / Uizard / v0 给一句话 → 出可编辑的 Figma 稿。Visily 截图 → wireframe 几分钟。

- **替代的是**:"按 brief 出 6 个 wireframe" 的体力活
- **没替代的是**:"用户在这一步会怎么想" 的同理心
- **设计师怎么用**:把 AI 当初稿引擎,自己 80% 时间花在交互细节 / a11y / 文案上

### 3. 视觉素材生产 → AI 90% 接管
图片(Flux / Midjourney)、视频(Sora / Kling)、3D(Meshy / Rodin)、配乐(Suno) —— 都能直接出可商用 quality。

- **替代的是**:中端摄影 / 中端剪辑 / 中端配乐外包
- **没替代的是**:艺术总监 + 顶级摄影师 + 艺术家(top 5% 反而更值钱)
- **设计师怎么用**:学一点 prompt engineering,把外包预算挪到 prompt 调试时间上

### 4. 用户研究 / 文案 → AI 50% 接管
Granola / Otter 自动转写访谈,Claude 总结要点 / 提炼 jobs-to-be-done。营销文案 / 微文案 AI 直接出。

- **替代的是**:转写 + 总结 + 第一版文案的"实习生工作"
- **没替代的是**:做访谈的同理心 + 文案的品牌人格
- **设计师怎么用**:产品经理岗位会和设计师并岗,文案 / 研究都得会一点

### 5. 设计系统 / 代码桥接 → AI 60% 接管
v0 / Galileo / Builder.io 让设计稿到代码不需要前端工程师手撸。

- **替代的是**:设计师"扔图给前端"的低效流程
- **没替代的是**:"系统怎么扩展" 的架构判断
- **设计师怎么用**:学会写一点 React / TypeScript,直接把设计 push 到 staging 让 PM 用

## 不被替代的 5 件事

1. **品牌判断** —— 这个像不像我们的品牌?这不是技能,是 taste
2. **跨职能协作** —— 跟 PM / 工程师 / 销售对齐目标,AI 不会代你开会
3. **取舍** —— 资源有限时哪个优先做?哪个不做?
4. **同理心** —— 知道用户为什么在这一步犹豫,而不是只看 UI
5. **审美 / 反审美** —— 在 AI 生成的"看起来都还不错"里挑出最对的那个

## 重新规划职业曲线

如果你是初级设计师(0-3 年),建议:

- 学一点工程(HTML / CSS / 一点 React),让你的设计能跑起来
- 每天用 AI 工具,把它们当成同事而不是工具
- 多做用户访谈 / 用户测试,这是 AI 短期完全做不了的

如果你是中级设计师(3-7 年),建议:

- 往产品 / 系统设计上走,做 design lead / staff designer
- 学会 prompt engineering(参考我们 [/prompts/midjourney-style-prompt](/prompts/midjourney-style-prompt))
- 培养跨职能影响力

如果你是高级 / lead(7+ 年),建议:

- 大概率不太担心,但要重新看团队组合 —— 你的下属里"产出型"职位会变少
- 学会用 AI 工具评估外包 / 候选人的 portfolio

## 不要焦虑,但要变

我自己是设计师出身。看着 Midjourney V7 的时候确实有过"我要被替代"的瞬间,但过半年后再看,**自己的价值反而在涨**。

差别在哪?**接受 AI 是同事,不是对手**。它做完概念探索,我做最终决策。它出 100 张图,我挑那 1 张。它写好 wireframe,我加上"用户在这里会怎么想"的细节。
